## 📖 快速上手

*注: Framix 的使用可能需要一定的命令行知识。如果您不熟悉命令行操作，建议先了解基础的命令行使用方法。*

#### 命令行参数启动
Framix 提供了一系列命令行参数来控制不同的功能和行为。

- **主要命令**
  - `--flick` : 循环模式。
  - `--paint` : 模拟绘制视频帧的分割线条。
  - `--video` : 分析指定的单个视频文件。
  - `--stack` : 分析整个视频文件夹。
  - `--merge` : 聚合多个时间戳报告。
  - `--union` : 聚合多个视频帧报告。
  - `--train` : 对视频文件进行构建模型前的归类。
  - `--build` : 构建图像处理模型。

- **附加命令**
  - `--alone` : 独立模式。
  - `--quick` : 快速模式。
  - `--basic` : 基础模式。
  - `--keras` : 智能模式。
  - `--boost` : 跳帧模式。
  - `--color` : 彩色模式。
  - `--shape` : 图片尺寸。
  - `--scale` : 缩放比例。
  - `--start` : 开始时间。
  - `--close` : 结束时间。
  - `--limit` : 持续时间。
  - `--crops` : 获取区域。
  - `--omits` : 忽略区域。

#### 使用示例

- 循环录制视频：
  ```
  framix --flick
  ```

- 循环录制分析视频帧并应用高级功能：
  ```
  framix --flick --keras --boost --color
  ```

- 循环录制视频，快速拆帧：
  ```
  framix --flick --quick
  ```
- 
- 循环录制视频，加载时间戳：
  ```
  framix --flick --basic
  ```

- 分析整个视频文件夹并应用高级功能：
  ```
  framix --keras --boost --color --stack <视频文件夹路径>
  ```
  
- 分析多个视频文件并应用高级功能：
  ```
  framix --keras --boost --color --crops 0,0.1,1,0.9 --video <视频文件路径-1> --video <视频文件路径-2>
  ```
  
- 模拟绘制视频帧的分割线条并应用高级功能：
  ```
  framix --paint --scale 0.4 --omits 0,0,1,0.2 --omits 0,0.9,1,0.1
  ```

- 为分析完毕的多个视频生成多元化测试报告：
  ```
  framix --merge <文件夹路径-1> --merge <文件夹路径-2> --merge <文件夹路径-3>
  ```
  
#### 高级功能
- **独立模式 (`--alone`)**: 循环模式下，单独控制每台设备的视频录制。
- **快速模式 (`--quick`)**: 对视频进行拆帧。
- **基础模式 (`--basic`)**: 对视频进行拆帧，加载时间戳。
- **智能模式 (`--keras`)**: 对视频进行拆帧，加载时间戳，AI自动分析视频帧。
- **跳帧模式 (`--boost`)**: 这个模式加速了视频处理的速度，适合快速预览和测试。
- **彩色模式 (`--color`)**: 用于生成彩色视频帧，提供更丰富的视觉效果。
- **图片尺寸 (`--shape`)**: 自定义视频帧的尺寸，以适应不同的分析要求。
- **缩放比例 (`--scale`)**: 自定义图片的缩放比例，以适应不同大小的屏幕。
- **开始时间 (`--start`)**: 在视频长度范围内选定视频的开始时间。
- **结束时间 (`--close`)**: 在视频长度范围内选定视频的结束时间。
- **持续时间 (`--limit`)**: 在视频长度范围内选定视频的持续时间。
- **开始帧 (`--begin`)**: 视频帧分类的开始阶段和开始帧。
- **结束帧 (`--final`)**: 视频帧分类结束阶段和结束帧。
- **获取区域 (`--crops`)**: 指定视频帧中应被获取的区域，这对于避免分析不必要的部分非常有用。
- **忽略区域 (`--omits`)**: 指定视频帧中应被忽略的区域，这对于避免分析不必要的部分非常有用。

#### 配置选项
- **新标题 (`header`)**: 为当前视频生成一个新的标题文件夹存放若干视频。
- **新设备 (`serial`)**: 从所有已连接的设备中重新选择设备。
- **新参数 (`deploy`)**: 在录制分析模式下修改配置文件以适应不同视频帧的分析策略。
- **新报告 (`create`)**: 生成汇总报告(时间戳视频帧)。
- **新报告 (`invent`)**: 生成汇总报告(帧编号视频帧)。
- **读秒数 (`******`)**: 支持最少 5 秒最大 300 秒的录制时长。

#### 多任务处理
Framix 融合了多进程、多线程和协程技术，提供了高效的视频处理能力。这些技术允许 Framix 同时分析多个视频或整个视频文件夹，大幅提升了处理速度。

- **多进程**: 利用 CPU 的多核优势，同时运行多个视频分析任务。
- **多线程**: 在单个进程中同时处理多个任务，优化资源使用和响应时间。
- **协程**: 提供更轻量级的并发处理，尤其适用于 I/O 密集型操作，如文件读写。

#### 交互式前端界面
Framix 提供了多个直观且功能强大的前端界面，帮助用户更容易地管理和分析视频。

- **可视化界面**: 清晰的 UI 设计，使得操作 Framix 更加直观和易懂。
- **交互式分析**: 通过前端界面，用户可以选择视频的开始帧和结束帧，以精确控制开始和结束部分。

#### 多阶段报告展示
Framix 允许用户将视频分析结果划分为多个阶段，便于对特定部分进行详细审查。

- **分阶段展示**: 用户可以将视频帧分解成多个阶段，例如启动阶段、交互阶段等，每个阶段的分析结果单独展示。
- **自定义阶段**: 根据需要，用户可自定义分阶段的标准和数量。

#### 重点帧展示
Framix 提供了突出显示视频中重要帧的功能。

- **重点帧识别**: 自动识别并突出显示视频中的关键帧，如性能峰值或异常帧。
- **重点帧导航**: 用户可以快速导航至这些重要帧，便于详细分析。

#### 综合报告汇总
Framix 支持将多个视频的分析结果汇总到一个综合报告中。

- **整体视角**: 提供全局视角，将多个视频的分析结果聚合展示。
- **对比分析**: 便于对比不同视频间的性能表现差异。

#### 自动帧差计算
Framix 能自动计算并展示视频帧之间的差异。

- **帧差展示**: 在报告中展示连续帧之间的变化，帮助识别性能波动点。
- **动态调整**: 用户可以根据需要调整帧差计算的精度和范围。

#### 一键复制功能
- **结果复制**: Framix 提供一键复制帧差结果的功能，便于将数据导出或分享。
- **快捷操作**: 提高了操作效率，尤其在需要将结果快速整合到其他文档或报告中时。

- 前端页面的展示效果优化，确保用户在分析和浏览结果时获得流畅的体验。
